MBS-fast считает рассеяние света несферическими ледяными частицами:
сначала трассируются геометрические лучи, затем для выходных пучков
считается дифракция в приближении Physical Optics. Код поддерживает
CPU-расчеты через OpenMP/MPI и CUDA-ускорение дифракции на GPU.

\subsection{Jones матрицы
лучей}\label{jones-ux43cux430ux442ux440ux438ux446ux44b-ux43bux443ux447ux435ux439}

Каждый пучок несет комплексную матрицу Jones \texttt{beam.J}:

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{J = [ Jvv  Jvh ]}
\NormalTok{    [ Jhv  Jhh ]}
\end{Highlighting}
\end{Shaded}

При отражениях/преломлениях \texttt{Splitting.cpp} домножает
\texttt{beam.J} на коэффициенты Fresnel в локальном
вертикальном/горизонтальном базисе луча. В PO режиме это амплитуда,
поэтому в обычном когерентном режиме сначала суммируются Jones матрицы
пучков, и только затем считается Mueller.

\subsection{Диады и поворот в базис

надо перевести в базис детектора. Это делают
\texttt{HandlerPO::RotateJones} и \texttt{RotateJonesFast}. Они строят
2x2 матрицу проекций через скалярные произведения между:

\end{longtable}

\end{Highlighting}
\end{Shaded}

элементы могут отличаться.

\subsection{Дифракционный интеграл
Kirchhoff}\label{ux434ux438ux444ux440ux430ux43aux446ux438ux43eux43dux43dux44bux439-ux438ux43dux442ux435ux433ux440ux430ux43b-kirchhoff}

Для каждого выходного пучка освещенная апертура представлена полигоном.
PO дальнее поле считается как интеграл по апертуре:

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{F(q) = integral\_A exp(i k q.r) dA}
\end{Highlighting}
\end{Shaded}

где \texttt{A} - полигон пучка, \texttt{k\ =\ 2*pi/lambda}, \texttt{q} -
разность между направлением пучка и направлением наблюдения. В коде
интеграл считается через границы полигона, а не через sampling площади.
Для ребра от \texttt{a} до \texttt{b} вклад выражается через разность
комплексных экспонент и фазовый знаменатель; для малых знаменателей
используются устойчивые sinc-пределы.

\begin{longtable}[]{@{}ll@{}}
\toprule\noalign{}
Функция & Назначение \\
\midrule\noalign{}
\endhead
\bottomrule\noalign{}
\endlastfoot

\subsection{Когерентная и некогерентная
сумма}\label{ux43aux43eux433ux435ux440ux435ux43dux442ux43dux430ux44f-ux438-ux43dux435ux43aux43eux433ux435ux440ux435ux43dux442ux43dux430ux44f-ux441ux443ux43cux43cux430}

По умолчанию:

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{J\_total(theta, phi) = sum\_beams J\_beam(theta, phi)}
\NormalTok{M(theta, phi)       = Mueller(J\_total(theta, phi))}
\end{Highlighting}
\end{Shaded}

С флагом \texttt{-\/-incoh}:

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{M(theta, phi) = sum\_beams Mueller(J\_beam(theta, phi))}
\end{Highlighting}
\end{Shaded}

Преобразование Jones -\textgreater{} Mueller реализовано в
\texttt{src/math/Mueller.cpp}: элементы 4x4 строятся из билинейных
комбинаций \texttt{\textbar{}S\_i\textbar{}\^{}2},
\texttt{Re(S\_i\ conj(S\_j))}, \texttt{Im(S\_i\ conj(S\_j))}.

\subsection{\texorpdfstring{Усреднение ориентаций и
\texttt{-\/-pole}}{Усреднение ориентаций и -\/-pole}}\label{ux443ux441ux440ux435ux434ux43dux435ux43dux438ux435-ux43eux440ux438ux435ux43dux442ux430ux446ux438ux439-ux438---pole}

Ориентационные режимы задают веса по beta/gamma. На точных beta-полюсах
все gamma эквивалентны, поэтому \texttt{-\/-pole} считает одну gamma и
умножает вес. В текущей версии при \texttt{-\/-pole} используются beta
endpoints, чтобы точка beta=0 или beta=pi действительно присутствовала в
сетке, а не заменялась midpoint.

\subsection{Веса scattering
grid}\label{ux432ux435ux441ux430-scattering-grid}

Для uniform theta grid выводится \texttt{Nth\ +\ 1} строк. Колонка
\texttt{2pi*dcos} - вес кольца телесного угла:

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{dOmega(theta\_j) = 2*pi * (cos(theta\_left) {-} cos(theta\_right))}
\end{Highlighting}
\end{Shaded}

Для endpoint строк используется половинная ячейка, обрезанная границей
диапазона. Для полной сетки \texttt{0..180} сумма всех \texttt{2pi*dcos}
должна быть \texttt{4*pi}.
