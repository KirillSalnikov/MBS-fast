\section{Overview}

MBS-fast computes light scattering by non-spherical particles using the Physical Optics (PO) method.
It traces geometric optics rays through the particle (reflections, refractions),
then applies Kirchhoff diffraction to each output beam to obtain the far-field Mueller matrix.

This manual describes the current code, including CPU, MPI/OpenMP, CUDA,
FFT angular interpolation, automatic averaging modes, cutoffs, checkpointing,
and the integral characteristics printed in the logs.  Formula names match the
variables in \texttt{src/handler/HandlerPOTotal.cpp},
\texttt{src/scattering/TracerPOTotal.cpp}, and
\texttt{src/cuda/GpuDiffraction.cu}.

\subsection{Method}

\begin{itemize}
\item \texttt{--po} --- Physical Optics (Kirchhoff diffraction). \textbf{Required for PO.}
\item \texttt{--go} --- Geometrical Optics (no diffraction).
\item \texttt{--incoh} --- Incoherent summation ($|J|^2$ per beam, then sum).

\section{Output Files and Console Log}

\subsection{Output files}

All output is written to the directory specified by \texttt{-o} (default \texttt{M}).

\subsubsection{\texttt{DIRNAME.dat} --- Mueller matrix}

Space-separated, 18 columns, one header line + one row per $\theta$ bin.

\begin{lstlisting}[basicstyle=\ttfamily\scriptsize]
ScAngle  2pi*dcos  M11  M12  M13  M14  M21  M22  M23  M24
                   M31  M32  M33  M34  M41  M42  M43  M44
\end{lstlisting}

\begin{itemize}
\item \texttt{ScAngle} --- scattering angle $\theta$ in degrees
\item \texttt{2pi*dcos} --- solid angle weight: $2\pi\,\Delta\cos\theta$.
  Integration: $C_\text{sca} = \sum_j M_{11}(\theta_j) \times (2\pi\,\Delta\cos\theta_j)$
\item \texttt{M11}\,\ldots\,\texttt{M44} --- 16 Mueller matrix elements (phi-averaged).
  For randomly-oriented particles: $M_{13}=M_{14}=M_{23}=M_{24}=M_{31}=M_{32}=M_{41}=M_{42}\approx 0$.
\end{itemize}

Useful derived quantities:
\begin{align*}
Q_\text{sca} &= C_\text{sca} / A_\text{proj} \\
g &= \frac{\sum M_{11}\cos\theta \cdot 2\pi\,d\cos\theta}{\sum M_{11}\cdot 2\pi\,d\cos\theta}
\qquad\text{(asymmetry parameter)} \\
-M_{12}/M_{11} &= \text{degree of linear polarization}
\end{align*}

\section{Current Implementation Reference}
\label{sec:implementation-reference}

This section is the exact operational model used by the current code.  It is
intended to answer ``what does this flag actually do'' rather than describe an
idealized method.

\subsection{Particle size, equivalent radius, and \texttt{--k\_eq}}

For every particle the code computes volume $V$, surface area $S$, maximal
dimension $D_{\max}$, equivalent-volume radius $r_\mathrm{eq}$, and equivalent
size parameter $k_\mathrm{eq}$:
\[
  r_\mathrm{eq} = \left(\frac{3V}{4\pi}\right)^{1/3},
  \qquad
  k_\mathrm{eq} = \frac{2\pi r_\mathrm{eq}}{\lambda}.
\]
\texttt{--rs SIZE} rescales a file particle so that
$D_{\max}=\texttt{SIZE}$.  \texttt{--k\_eq K} rescales it so that
$k_\mathrm{eq}=K$.  The scale ratio is
\[
  a = \frac{K\lambda/(2\pi)}{r_\mathrm{eq,current}},
  \qquad
  D_{\max,new}=aD_{\max,current}.
\]
The log prints both \texttt{r\_eq} and \texttt{k\_eq}; this is the same
definition used in ADDA-style comparisons.

\subsection{Mueller output and azimuth averaging}

For each output scattering angle $\theta_j$, the code stores a full
$N_\varphi\times N_\theta$ Mueller accumulator $M(p,j)$.  The written
one-dimensional matrix is an azimuth average.  Away from the poles it applies
the rotation matrix \texttt{Lp}:
\[
  L_p(\varphi)=
  \begin{pmatrix}
  1&0&0&0\\
  0&\cos2\varphi&\sin2\varphi&0\\
  0&-\sin2\varphi&\cos2\varphi&0\\
  0&0&0&1
  \end{pmatrix},
\]
\[
  \bar M(\theta_j)
  =\frac{1}{N_\varphi}\sum_{p=0}^{N_\varphi-1}
     M(p,j)L_p(\varphi_p).
\]
At $\theta=0$ and $\theta=\pi$ the code first averages without $L_p$ and then
projects onto the pole-symmetric Mueller form using
\texttt{ApplyForwardPoleSymmetry()} or \texttt{ApplyBackwardPoleSymmetry()}.

\subsection{Solid-angle weights}

The second column in every \texttt{.dat} file is exactly the value returned by
\texttt{ScatteringRange::Compute2PiDcos(j)}.  For a uniform theta grid with
step $\Delta\theta$:
\[
  w_j = 2\pi\,\Delta\cos_j.
\]
For pole rows:
\[
  w_0 = w_N = 2\pi\left(1-\cos\frac{\Delta\theta}{2}\right).
\]
For interior rows:
\[
  w_j = 2\pi\left[
    \cos\left((j-\tfrac12)\Delta\theta\right)
    -\cos\left((j+\tfrac12)\Delta\theta\right)
  \right].
\]
For a non-uniform \texttt{--tgrid}/\texttt{--auto\_tgrid} grid, bin boundaries
are midpoints between neighboring theta values:
\[
  w_j = 2\pi\left(\cos\theta_{j,-}-\cos\theta_{j,+}\right).
\]

\subsection{Integral characteristics}
\label{sec:integral-characteristics}

The following quantities are printed by \texttt{HandlerPOTotal::WriteMatricesToFile}
and \texttt{WriteAveragedRowsFile}.  They are intentionally separated because
they answer different questions.

\paragraph{Projected area / incoming energy.}
\[
  A_\mathrm{proj} =
  \sum_{\mathrm{orientations}} A_\mathrm{illuminated}(\Omega_i)\,w_i.
\]
This is printed as \texttt{A\_proj (incoming energy)} and is the denominator
for all efficiencies:
\[
  Q_x=\frac{C_x}{A_\mathrm{proj}}.
\]

\paragraph{Angular scattering integral.}
\[
  C_{\mathrm{sca,integral}} =
    \sum_j \bar M_{11}(\theta_j)\,w_j.
\]
This is a numerical angular integral of the written $M_{11}$ curve.  It is
very sensitive to the theta grid near a narrow forward peak.  On a coarse grid
it can be much larger or smaller than the optical-theorem value; the code then
prints it as a diagnostic rather than silently using it as a physical truth.

\paragraph{Geometrical-optics energy balance.}
If absorption accounting is enabled, the code also accumulates outgoing beam
energy from traced beams:
\[
  C_{\mathrm{abs,GO}} = A_\mathrm{proj}-E_\mathrm{out}.
\]
If absorption is not enabled, the code sets
$C_{\mathrm{abs,GO}}=0$.  This is printed as \texttt{C\_abs\_GO}; it is a
diagnostic of the ray/beam energy balance and is not used to overwrite the
optical-theorem extinction.

\paragraph{Optical-theorem extinction.}
For each prepared orientation the code evaluates the forward diffraction
amplitude and accumulates
\[
  C_{\mathrm{ext,OT}} =
    \lambda\,\operatorname{Im}(f_{00}+f_{11})\,w_i,
\]
where $w_i$ is the orientation weight stored as
\texttt{PreparedOrientation::sinZenith}.  The global printed value is the sum
over all orientations.  The implementation is
\texttt{HandlerPO::ComputeForwardExtinctionOt()}.
The default amplitude is the object-centered spherical-wave amplitude $S$ in
the convention of Gao/Yang/Kattawar Eq. (9), not the far-screen field
$\tilde S$.  Therefore no hidden far-zone phase correction is applied in the
default optical theorem.  The diagnostic flag \texttt{--ot\_ping D} only
reproduces legacy files whose saved forward amplitude still contained the
far-screen phase:
\[
  \operatorname{Im}S \rightarrow
  \operatorname{Im}S\cos(2kD)-\operatorname{Re}S\sin(2kD).
\]

\paragraph{Reported cross sections.}
The final reported values are:
\[
  C_{\mathrm{ext,legacy}} =
    C_{\mathrm{sca,integral}} + C_{\mathrm{abs,GO}}.
\]
If optical-theorem extinction is available:
\[
  C_\mathrm{ext}=C_{\mathrm{ext,OT}}.
\]
If OT is not available:
\[
  C_\mathrm{ext}=C_{\mathrm{ext,legacy}}.
\]
For absorbing particles ($\operatorname{Im}m\ne0$):
\[
  C_\mathrm{sca}=C_{\mathrm{sca,integral}},
  \qquad
  C_\mathrm{abs}=C_{\mathrm{ext,OT}}-C_{\mathrm{sca,integral}}.
\]
For non-absorbing particles ($\operatorname{Im}m=0$) and available OT:
\[
  C_\mathrm{abs}=0,
  \qquad
  C_\mathrm{sca}=C_{\mathrm{ext,OT}}.
\]
The angular integral remains in the log as
\texttt{Csca\_integral}.  If
$|C_{\mathrm{sca,integral}}-C_{\mathrm{ext,OT}}|/|C_{\mathrm{ext,OT}}|>1\%$,
the code prints a mismatch warning.  This is expected on too-coarse theta grids.

\subsection{Physical optics diffraction per beam}

For each geometric beam the code stores a Jones matrix $J$, polygon area,
edge data, local polarization bases, optical path, and projected length.  The
far-field Jones contribution is a product of:
\[
  J_\mathrm{sca}(\theta,\varphi)
  = R(\theta,\varphi)\,F_\mathrm{Kirchhoff}(\theta,\varphi)\,
    e^{i\Phi(\theta,\varphi)}\,J_\mathrm{beam}.
\]
The hot loop evaluates the polygon Kirchhoff integral through precomputed edge
slopes.  If both phase coefficients $A$ and $B$ are near zero, the analytic
limit is used:
\[
  F_\mathrm{Kirchhoff}
  = \pm \frac{i}{\lambda}\,A_\mathrm{beam},
\]
with the current default sign chosen by the forward-direction bugfix.  The
\texttt{--legacy\_sign} flag restores the older sign convention.

The Jones-to-Mueller conversion in \texttt{AddToMuellerLocal()} uses the code's
explicit component formulas.  For example:
\[
  M_{11}=\frac12\left(|J_{00}|^2+|J_{10}|^2
                    +|J_{01}|^2+|J_{11}|^2\right).
\]
Default mode sums beams coherently within one orientation and then converts to
Mueller.  \texttt{--incoh} converts each beam to Mueller first and sums
intensities.  \texttt{--coh\_orient} is a legacy mode that coherently sums
Jones amplitudes across all orientations before converting; it is not the
default physical orientation average.

\subsection{Orientation weights and averaging modes}

\paragraph{\texttt{--fixed BETA GAMMA}.}
Runs one body orientation.  Angles are degrees.

\paragraph{\texttt{--random Nb Ng}.}
Uses a regular beta/gamma product grid over the particle symmetry domain.
The current oldauto/random path uses beta midpoints by default:
\[
  \beta_i = \beta_{\min} + (i+\tfrac12)\Delta\beta,
\]
unless environment variable \texttt{MBS\_OLDAUTO\_BETA\_MIDPOINT=0} is set.
Gamma points are
\[
  \gamma_j=\gamma_{\min}+j\Delta\gamma,
\]
with an optional deterministic stagger in oldauto/random.  The beta weight is
computed by \texttt{CalcCsBeta()} from the cosine interval, and gamma is
normalized by the number of gamma samples and the beta symmetry factor.

\paragraph{\texttt{--oldauto DIV}.}
This is the production regular-grid mode used for strict ADDA comparisons.
It derives orientation density from a diffraction angular step:
\[
  \Delta\theta_\mathrm{diff}
    = 0.69\,\frac{\lambda}{D_{\max}}\frac{180}{\pi}
    \quad\mathrm{degrees}.
\]
The orientation step is
\[
  \Delta\theta_\mathrm{orient}
    = \frac{\Delta\theta_\mathrm{diff}}{\texttt{--ring\_points}},
  \qquad \texttt{--ring\_points default}=3.
\]
For particle symmetry ranges $\beta_\mathrm{sym}$ and $\gamma_\mathrm{sym}$:
\[
  N_{\beta,\mathrm{full}}
    = \left\lceil\frac{\beta_\mathrm{sym}[\deg]}
                       {\Delta\theta_\mathrm{orient}}\right\rceil,
  \quad
  N_{\gamma,\mathrm{full}}
    = \left\lceil\frac{\gamma_\mathrm{range}[\deg]}
                       {\Delta\theta_\mathrm{orient}}\right\rceil.
\]
\[
  N_\beta = \max\left(3,\left\lceil
      \frac{N_{\beta,\mathrm{full}}}{\texttt{DIV}}\right\rceil\right),
  \quad
  N_\gamma = \max\left(3,\left\lceil
      \frac{N_{\gamma,\mathrm{full}}}{\texttt{DIV}}\right\rceil\right).
\]
\texttt{--oldauto 2} is stricter than \texttt{--oldauto 4}; \texttt{--oldauto 1}
is the reference-level regular grid.

\paragraph{\texttt{--sobol N}, \texttt{--sobol\_seed N S}.}
Uses a Sobol low-discrepancy sequence on the symmetry-reduced orientation
domain.  \texttt{--sobol} uses seed 42.  \texttt{--sobol\_seed} uses an
explicit nested Owen scramble seed.  Output weights are equal after mapping to
the symmetry domain.

\paragraph{\texttt{--sobol\_ring Nb Ng}.}
Uses a Sobol-like beta distribution combined with a uniform shifted gamma ring.
This is useful when gamma periodicity aliases specular events.

\paragraph{\texttt{--hammersley}, \texttt{--lattice}, \texttt{--lattice\_z}.}
Alternative low-discrepancy orientation rules.  \texttt{--lattice\_z N Z}
uses explicit rank-1 generator $Z$; \texttt{--lattice N} normalizes/selects
the generator internally.

\paragraph{\texttt{--euler\_quad Nb Ng}.}
Uses high-order quadrature in $\cos\beta$ and periodic gamma points.  It is
slower than Sobol for broad scans but useful as a deterministic quadrature
reference.

\paragraph{\texttt{--adaptive EPS}.}
Starts from an oldauto-based estimate, evaluates control quantities, and
increases Sobol orientation count until relative changes are below EPS or the
oldauto-derived cap is reached.  The control includes the angular integral and
selected $M_{11}$ rows, especially near backscatter.

\paragraph{\texttt{--auto EPS}.}
Convenience mode: implies adaptive theta grid, auto phi grid, and adaptive
Sobol orientations.  It does not use a final regular oldauto grid.

\paragraph{\texttt{--autofull EPS}.}
Sequentially tunes:
\[
  n \;\rightarrow\; N_\varphi \;\rightarrow\; N_\mathrm{orient}.
\]
It is intended to search parameters, not to be the fastest production mode.
It writes maps such as \texttt{\_autofull\_phi\_by\_theta.dat} and
\texttt{\_autofull\_orient\_by\_theta.dat}.

\paragraph{\texttt{--oldautofull EPS}.}
Uses the autofull search but finishes with a regular oldauto-style
$\beta\times\gamma$ grid.  This was added because irregular Sobol/adaptive
orientation sets interact poorly with mirror-sector reuse and some multigrid
comparisons.  The final orientation count is selected by
\texttt{AutoFullOldautoEstimateForTarget()} and is at least an oldauto div4
grid in strict mode.

\paragraph{\texttt{--owen\_avg K}, \texttt{--owen\_seeds S...}.}
Only affects \texttt{--autofull} final averaging.  It repeats the final Sobol
average over $K$ nested Owen seeds or over the explicit seed list.

\subsection{Symmetry and mirror gamma}

Particle classes provide a symmetry domain:
\[
  \beta\in[0,\beta_\mathrm{sym}],
  \qquad
  \gamma\in[0,\gamma_\mathrm{sym}].
\]
\texttt{--sym Sb Sg} overrides this as:
\[
  \beta_\mathrm{sym}=\pi/S_b,\qquad
  \gamma_\mathrm{sym}=2\pi/S_g.
\]
\texttt{--mirror\_gamma} halves the gamma domain:
\[
  \gamma_\mathrm{range}=\gamma_\mathrm{sym}/2.
\]
If it controls a regular gamma grid, the code rounds the gamma count to an
even number.  If \texttt{--mirror\_gamma} or \texttt{--fft} is active,
$N_\varphi$ is rounded upward to a multiple of 6.  This is required for exact
hexagonal phi-sector mapping; direct non-FFT calculations merely warn when
$N_\varphi\bmod6\ne0$.

\subsection{Scattering theta/phi grid flags}

\paragraph{\texttt{--grid T1 T2 Nphi Ntheta}.}
Uniform full theta interval in degrees.  The fourth argument is the number of
theta intervals, so output has $N_\theta+1$ rows.  Example
\texttt{--grid 0 180 600 180} writes 181 theta rows.

\paragraph{\texttt{--grid R Nphi Ntheta}.}
Backscatter cone grid from $\theta=\pi-R$ to $\theta=\pi$.

\paragraph{\texttt{--tgrid FILE}.}
Reads non-uniform theta values in degrees, one per line.  These values become
the exact output rows and the integration weights are midpoint-cell weights.

\paragraph{\texttt{--auto\_tgrid EPS}.}
Performs recursive bisection using pilot $M_{11}$ values.  For an interval
$(\theta_a,\theta_b)$ it computes the midpoint and compares it with linear
interpolation:
\[
  e = \frac{|M_{11}(\theta_m)-M_{11}^{lin}(\theta_m)|}
           {\max(1, |M_{11}(\theta_m)|, |M_{11}^{lin}(\theta_m)|)}.
\]
If $e>\texttt{EPS}$ and the maximum depth/step limits allow it, the interval is
split.  The final full run uses the resulting non-uniform theta grid.

\paragraph{\texttt{--auto\_phi}, \texttt{--nphi}.}
\texttt{--nphi} has highest priority.  \texttt{--auto\_phi} chooses an
internally estimated value and then applies the multiple-of-6 rule when needed.

\subsection{CUDA backend}

\texttt{--gpu} moves Phase~2 diffraction to CUDA.  The CPU still performs:
particle rotations, shadow construction, geometric tracing, beam splitting,
beam filtering, and prepared-beam packing.  CUDA processes prepared beams over
the output angular grid.  The GPU build has two precision variants:
\begin{itemize}
\item \texttt{gpu/bin/mbs\_po\_gpu\_float\_fast}: single-precision CUDA
  diffraction; fastest on consumer GPUs.  The near-singularity epsilon is
  clamped to at least $3\times10^{-5}$ in \texttt{MBS\_GPU\_FLOAT}.
\item \texttt{gpu/bin/mbs\_po\_gpu\_double\_fast}: double-precision CUDA
  diffraction; slower on consumer GPUs, often competitive on V100-class cards.
\end{itemize}
If CUDA errors occur the default behavior is fatal.  Set
\texttt{MBS\_GPU\_ALLOW\_FALLBACK=1} only for debugging, because then a failed
GPU calculation can silently continue on CPU.

\subsection{FFT angular interpolation}

\texttt{--fft} is an angular Fourier interpolation in output azimuth
$\varphi$, not a spatial pFFT/FMM.  The code computes diffraction directly on a
reduced uniform phi grid of size
\[
  N_{\varphi,low} = N_{\varphi,full}/F,
\]
transforms rows by cuFFT, zero-pads Fourier modes, inverse-transforms to
$N_{\varphi,full}$, and adds the interpolated Mueller arrays to the final
accumulator.

Default factor selection in \texttt{choose\_fft\_phi\_factor()}:
\[
F=\begin{cases}
8, & N_{\varphi,full}\ge2400,\\
6, & N_{\varphi,full}\ge600,\\
3, & N_{\varphi,full}\ge240,\\
2, & \text{otherwise.}
\end{cases}
\]
\texttt{MBS\_FFT\_PHI\_FACTOR=N} overrides this.  \texttt{MBS\_FFT\_THETA\_FACTOR}
can enable experimental theta interpolation for uniform compatible grids.

Quality/debug environment variables:
\begin{itemize}
\item \texttt{MBS\_FFT\_CHECK=1}: also computes a denser/direct reference and
  prints max/RMS relative errors.  This is a diagnostic and costs time.
\item \texttt{MBS\_FFT\_ADAPTIVE\_PHI=1}: detects suspicious theta rows after
  FFT interpolation and recomputes selected rows directly.
\item \texttt{MBS\_FFT\_ADAPTIVE\_PHI\_THRESHOLD}: default 0.04.
\item \texttt{MBS\_FFT\_ADAPTIVE\_PHI\_MAX\_ROWS}: default 8.
\item \texttt{MBS\_GPU\_FFT\_PAIR=1}: interpolates full and no-shadow arrays as
  a pair to reduce duplicated FFT setup.
\end{itemize}

For non-convex CPU tracing, a conservative projected bounding-box prefilter is
enabled by default before the exact polygon intersection. Set
\texttt{MBS\_TRACE\_CPU\_PREFILTER=0} to disable it. A beam is projected onto
the candidate facet plane along its propagation direction; if the expanded 2D
boxes do not overlap, the exact intersection is skipped.
\texttt{MBS\_TRACE\_CPU\_PREFILTER\_MARGIN} sets the expansion margin; the
default is 8 in particle length units. This default is intentionally
conservative; reduce it only after checking the Mueller matrix against a
no-prefilter reference for the specific particle.
\texttt{MBS\_TRACE\_TREE\_RESERVE} controls the initial
\texttt{std::vector<Beam>} capacity for the non-convex beam stack. The default
is 16384, which reduces reallocations and beam copies in deep non-convex
traces. Lower it only if memory is tight; raise it if logs show much larger
per-orientation beam counts.
For non-absorbing particles (\(\Im m=0\)) BigInteger trace IDs are skipped by
default because absorption path recovery does not use them; set
\texttt{MBS\_FORCE\_TRACK\_IDS=1} for debug or explicit track-output workflows.

\texttt{--gpu\_trace} is not completed full GPU ray tracing. It is only an
experimental CUDA broad-phase candidate prefilter for non-convex tracing:
candidate beam/facet pairs may be projected and rejected on the GPU, but all
surviving pairs still use the exact CPU polygon intersection. For large
non-convex particles, Phase~1 can still be the main bottleneck because
geometric tracing, beam splitting, internal-reflection expansion, and exact
intersections remain CPU-side. The production CUDA backend accelerates
diffraction, not the full tracing stack.

The CUDA prefilter is intentionally conservative in production builds:
\texttt{MBS\_GPU\_TRACE\_MIN\_CANDIDATES} defaults to 65536, and with
OpenMP tracing (\texttt{--threads > 1}) it is disabled unless
\texttt{MBS\_GPU\_TRACE\_OPENMP=1}.  This avoids the pathological case where
many CPU threads launch small CUDA prefilter batches and serialize Phase~1.

\subsection{Beam and trace cutoffs}

Beam cutoff happens after tracing, before diffraction.  For one orientation the
code computes:
\[
  J_{\max}=\max_b \|J_b\|^2,\qquad
  A_{\max}=\max_b A_b,\qquad
  I_{\max}=\max_b(\|J_b\|^2 A_b).
\]
A prepared beam is skipped if any enabled test is true:
\[
  \|J_b\|^2 < \epsilon_J J_{\max},
  \quad
  A_b < \epsilon_A A_{\max},
  \quad
  \|J_b\|^2A_b < \epsilon_I I_{\max}.
\]
Flags:
\begin{itemize}
\item \texttt{--beam\_cutoff EPS}: shorthand for the J and area thresholds
  unless the separate flags override them.
\item \texttt{--beam\_cutoff\_j EPS}: sets $\epsilon_J$.
\item \texttt{--beam\_cutoff\_area EPS}: sets $\epsilon_A$.
\item \texttt{--beam\_cutoff\_importance EPS}: sets $\epsilon_I$.
\end{itemize}
Trace cutoff happens earlier while expanding internal beams.  It has analogous
flags \texttt{--trace\_cutoff\_j}, \texttt{--trace\_cutoff\_area}, and
\texttt{--trace\_cutoff\_importance}; \texttt{--trace\_cutoff EPS} sets the J
and area tests.  \texttt{--trace\_max\_beams N} aborts expansion for an
orientation after $N$ traced beam nodes.  Cutoffs are speed/accuracy tradeoffs;
for strict reference runs use zero cutoffs.

\subsection{Multigrid and multikeq}

\texttt{--multigrid Dmin Dmax N} generates $N$ logarithmically spaced $D_{\max}$
values.  \texttt{--multikeq Kmin Kmax N} generates logarithmically spaced
$k_\mathrm{eq}$ values.  \texttt{--multikeq\_list FILE} uses exact values from
the file.

With \texttt{--oldauto}, the particle is first resized to the largest requested
size.  The largest size determines the oldauto orientation grid and the shared
beam cache.  Smaller sizes reuse the traced topology and rescale diffraction
phases.  This is fast, but for non-convex particles it assumes that the
reference-size beam topology is acceptable for all smaller sizes.  For maximum
robustness use separate single-size runs or \texttt{--multigrid\_parallel}.

\texttt{--multigrid\_parallel J} starts up to $J$ child processes.  In GPU
workflows this is useful for filling one GPU with multiple independent small
jobs, but two heavy jobs on one GPU can reduce throughput.  Use
\texttt{--multigrid\_threads T} to set OpenMP threads per child.

\subsection{Checkpointing}

\texttt{--checkpoint} is opt-in.  For \texttt{--orientfile} it writes
\texttt{\_checkpoint.bin} after each chunk.  For \texttt{--oldauto}/
\texttt{--random} it writes \texttt{\_oldauto\_checkpoint.bin} after each
completed beta ring.  The checkpoint stores accumulated Mueller arrays,
incoming energy, outgoing energy, OT extinction, flags, and a parameter hash.
On restart with the same \texttt{-o} path and matching hash the run resumes
from the next beta ring.  Oldauto/random checkpointing is disabled under MPI.

\subsection{Threading, MPI, and scheduling}

\texttt{--threads N} sets OpenMP worker threads.  By default the program sets
OpenMP to physical cores, not hyperthreads.  In \texttt{--oldauto --gpu} runs,
CPU threads trace gamma chunks while the GPU diffracts prepared chunks.  The
log line
\begin{lstlisting}
Oldauto/random memory: gamma chunk=16/303 per beta
\end{lstlisting}
means that only 16 gamma orientations are prepared at a time for that beta ring.
\texttt{--chunk N} overrides this chunk size.  MPI, when enabled in the CPU
binary, splits beta rings across ranks for oldauto/random and reduces Mueller
arrays and energies at the end.

\subsection{Key optimizations}
\begin{enumerate}
\item \textbf{PrecomputeEdgeData}: slopes, intercepts computed once per beam (not per direction).
\item \textbf{Theta-coefficients}: $A(\theta) = \sin\theta\cdot a_s + \cos\theta\cdot a_c + a_0$ (2 FMA).
\item \textbf{Batched sincos}: all vertex phases for all $\theta$ pre-computed before the $\theta$-loop. AVX-512 \texttt{fast\_sincos\_8x} or AVX2 $2\times$\texttt{fast\_sincos\_4x}.
\item \textbf{Polynomial sincos}: Cody--Waite range reduction + Chebyshev polynomial ($\sim$14 decimal digits).
\item \textbf{Inline hot loop}: no heap allocation, stack-only \texttt{complex} arithmetic.
\item \textbf{OpenMP}: thread-local Mueller accumulators, \texttt{schedule(dynamic,1)}.
\item \textbf{Sobol + symmetry}: native C++ Sobol generator with particle symmetry reduction.
\end{enumerate}

\section{CUDA / FFT Recommendations for File Particles}

For large file particles and shared size sweeps, use the CUDA diffraction
backend with FFT phi interpolation and the global orientation scheduler:

\begin{lstlisting}
MBS_SHARED_PIPELINE=1 MBS_SHARED_ORIENT_CHUNK=64 \
./bin/mbs_po_float_fast --pf Afine30.dat --multikeq_list keq_list.txt \
    --ri 1.6 0.002 -n 14 --po --oldauto 2 \
    --beam_cutoff_j 0.001 --beam_cutoff_area 0.002 \
    --trace_cutoff_importance 0.0001 --trace_max_beams 20000 \
    --grid 0 180 600 181 -w 1.064 --gpu --fft --close
\end{lstlisting}

\subsection{\texttt{--gpu --fft}}

\texttt{--gpu} moves the Phase 2 diffraction integral to CUDA. The CPU still
performs ray tracing and prepares beam data. \texttt{--fft} enables angular
Fourier interpolation in $\varphi$: diffraction is evaluated on a reduced direct
azimuth grid and reconstructed to the requested $N_\varphi$ by cuFFT. This is
not aperture pFFT/FMM and not a 3D spatial FFT.

Recommended scattering grid for ADDA comparisons used here:
\begin{lstlisting}
--grid 0 180 600 181
\end{lstlisting}
This gives 600 azimuth points and 181 theta points, including 0 and 180 degrees.

\subsection{\texttt{MBS\_SHARED\_ORIENT\_CHUNK=N}}

Shared \texttt{--oldauto --multikeq*} GPU runs use a global orientation
scheduler. Orientations are indexed as
\[
  i = i_\beta N_\gamma + i_\gamma .
\]
The scheduler traces contiguous chunks of \texttt{N} orientations and
immediately sends each completed chunk to CUDA diffraction. This avoids waiting
for a whole beta block when one non-convex orientation is slow.

Default for GPU shared multikeq:
\begin{lstlisting}
MBS_SHARED_ORIENT_CHUNK=64
\end{lstlisting}

Memory scales approximately as
\[
  M_\mathrm{host} \sim N_\mathrm{chunk}
      \langle N_\mathrm{beams/orientation}\rangle ,
\]
instead of
\[
  M_\mathrm{host} \sim N_\mathrm{betaGroup} N_\gamma
      \langle N_\mathrm{beams/orientation}\rangle .
\]

Practical choices:
\begin{itemize}
\item \texttt{MBS\_SHARED\_ORIENT\_CHUNK=64}: recommended default for
  Afine30 / Greek file-particle runs.
\item \texttt{MBS\_SHARED\_ORIENT\_CHUNK=32}: lower latency; use when GPU
  utilization has long zero-utilization gaps.
\item \texttt{MBS\_SHARED\_ORIENT\_CHUNK=128}: fewer CUDA launches; test when
  GPU launch overhead dominates and host RAM is available.
\end{itemize}

On Afine30 with \texttt{--grid 0 180 600 181 --gpu --fft --oldauto 2}, chunks
of 32--64 improved the observed scheduler rate from about 4.3 orientations/s
to about 5.5 orientations/s on the RTX 4070 SUPER node and reduced host RAM
from roughly 10 GB to below 2 GB.

\subsection{\texttt{MBS\_SHARED\_PIPELINE=1}}

This overlaps CPU tracing of chunk $k+1$ with CUDA diffraction of chunk $k$.
Use it with the global scheduler:
\begin{lstlisting}
MBS_SHARED_PIPELINE=1 MBS_SHARED_ORIENT_CHUNK=64
\end{lstlisting}

If the GPU is still idle for long intervals, reduce the chunk to 32. If the GPU
is busy but many tiny launches dominate, increase it to 128.

\subsection{\texttt{MBS\_SHARED\_BETA\_GROUP=N}}

This is the older beta-block batching control. Large values reduce CUDA launch
count but can be slower on non-convex particles because one hard orientation
holds the whole beta group. With the global scheduler enabled, treat this as a
diagnostic/legacy knob. Use:
\begin{lstlisting}
MBS_SHARED_BETA_GROUP=1  # lowest latency
MBS_SHARED_BETA_GROUP=2  # conservative beta-block test
MBS_SHARED_BETA_GROUP=4  # larger packet, more RAM, longer tails
\end{lstlisting}

Recommended production mode is the global orientation chunk, not large beta
groups.

\subsection{\texttt{MBS\_GPU\_MULTI\_K\_FULL=1}}

This is an opt-in fused multi-\(k_\mathrm{eq}\) CUDA path for
\texttt{--multikeq\_shared\_batches}.  The ordinary shared-batch mode traces
and prepares beams once, but still packs and diffracts each target size
separately.  With
\begin{lstlisting}
MBS_GPU_MULTI_K_FULL=1
\end{lstlisting}
the CUDA backend packs the reference prepared beams once and launches a kernel
whose output layout is
\[
  M[s, p, t, 16],
\]
where \(s\) indexes the target size.  For each target size,
\[
  a_s = k_{\mathrm{eq},s}/k_{\mathrm{eq},ref}
      = D_s/D_\mathrm{ref}.
\]
The kernel evaluates the same diffraction integral as the per-size path by
using
\[
  k_s = a_s k,\qquad k_s^2 = a_s^2 k^2,
\]
instead of explicitly copying scaled coordinates.  The raw Jones matrix and
unscaled phase path are stored in the GPU beam and phased as
\[
  J_s =
      \sigma\,J_0\,\exp(i k a_s L),
\]
where \(\sigma\) is the existing external/odd-reflection sign.  In the
singular forward integral branch, the aperture-area result is scaled by
\(a_s^2\), matching the old path where \(\text{area}_s=a_s^2\text{area}\).

For absorbing beams, internal path samples are copied to CUDA in a compact
offset/value array.  The kernel applies the same averaged absorption factor as
the CPU packing code:
\[
  A_s =
    \frac{1}{N}\sum_i \exp(c_\mathrm{abs}\,l_i\,a_s),
\qquad
  J_s \leftarrow A_s J_s.
\]

The fused path is deliberately conservative.  It falls back to the ordinary
per-size CUDA path for no-shadow output and for FFT check/refine modes.  With
\texttt{--fft}, it fuses only the direct low-\(N_\varphi\) pass and then runs
the normal cuFFT interpolation independently for each target size.

Short V100 checks with \texttt{--fft}, absorbing beams, and three nearby
sizes showed Phase~2 speedup from \(0.59\)~s to \(0.23\)~s, with maximum
absolute Mueller differences of order \(3\times10^{-6}\) versus the old
per-size GPU path.

\section{Bugfixes}

\subsection{Forward-direction Fresnel sign (2026-03-22)}

\textbf{Bug}: sign error in the forward-direction special case ($A{\approx}0, B{\approx}0$) of the Kirchhoff integral.
The edge-sum converges to $-F_\text{inv}\cdot\text{area}$ but the shortcut used $+F_\text{inv}\cdot\text{area}$.

\textbf{Symptom}: $M_{11}(0^\circ)$ anomalously small in coherent mode for small particles.
Example (hex $D{=}H{=}3\,\mu$m): $M_{11}(0)/M_{11}(0.1^\circ) = 0.019$ (should be $\approx 1.03$).

\textbf{Fix}: negate the forward-direction formula in 6 locations:
\begin{itemize}
\item \texttt{HandlerPO.cpp}: lines 970, 1333, 1889
\item \texttt{Handler.cpp}: lines 255, 473, 642
\end{itemize}

Incoherent mode unaffected ($|J|^2$ is sign-independent).
$Q_\text{sca}$ changed by $\sim$0.06\%.
See \texttt{BUGFIX\_forward\_direction\_sign.md} for full details.

\section{Known Limitations}

\begin{enumerate}
\item \textbf{$Q_\text{sca} > 2$ at large $x$}: PO does not enforce the optical theorem.
\item \textbf{$M_{33}, M_{34}, M_{44}$}: RotateJones vs Karczewski give different off-diagonal elements (same $M_{11}$).
\item \textbf{Small particles ($x < 20$)}: PO not valid. Use ADDA or T-matrix.
\item \textbf{Backscattering convergence}: $M_{11}(180^\circ)$ needs 10--100$\times$ more orientations.
\item \textbf{LTO}: \texttt{-flto} causes $\sim$20\% regression. Do not use.
\end{enumerate}

\section{Bugfix: 15$\times$ Mismatch (old vs new binary)}
\label{sec:bugfix15x}

Comparison of \texttt{--fixed 45 30} between old and new binaries showed a $15.5\times$ difference in $M_{11}$.

\subsection{Root causes}

\begin{enumerate}
\item \textbf{Old \texttt{--fixed} never rotated the particle.}
  \texttt{ScatteringConvex::ScatterLight()} has \texttt{m\_particle->Rotate()} commented out,
  and old \texttt{TracerPO::TraceFixed} did not call it either.
  Result: \texttt{--fixed 45 30} always gave the default orientation.
  New code correctly adds \texttt{m\_particle->Rotate(b, g, 0)} in \texttt{TraceFixed}.

\item \textbf{Uninitialized \texttt{d\_param} in particle constructors.}
  \texttt{Point3f()} does not zero \texttt{coordinates[3]}.
  No constructor called \texttt{SetDParams()} after \texttt{Reset()}.
  Fixed: \texttt{SetDParams()} added after \texttt{Reset()} in all 8 particle constructors.

\item \textbf{Mixed \texttt{fast\_exp\_im}/\texttt{exp\_im} in fallback paths.}
  \texttt{DiffractIncline}, \texttt{DiffractInclineFast}, \texttt{ComputeFnJones},
  and \texttt{ApplyDiffraction*} had inconsistent precision.
  Fixed: all fallback paths use \texttt{exp\_im} (glibc sincos).
\end{enumerate}

\subsection{Validation}

\begin{tabular}{lrll}
\toprule
Test & Max diff & Mode & Notes \\
\midrule
\texttt{--fixed 0 0 --incoh} & exact & incoh & per-beam $|J|^2$ identical \\
\texttt{--random 64 43 --incoh} & exact & incoh & per-beam physics identical \\
\texttt{--random 32 22 --coh\_orient} & $10^{-7}$ & coh & machine precision \\
\texttt{--random 64 43} (default) & $0.007\%$ & incoh avg & coh vs incoh averaging \\
HandleBeams inline vs fallback & identical & & SIMD optimizations verified \\
\bottomrule
\end{tabular}

\medskip
\textbf{Speedup:} 55$\times$ (23\,min $\to$ 25\,sec at 2860 orientations, 4 threads).

\textbf{d\_param note:}
\texttt{d\_param} is the plane equation parameter ($ax+by+cz+d=0$) stored in
\texttt{Point3f::coordinates[3]}. It was uninitialized before \texttt{Rotate()} call.
\texttt{SetDParams()} now added to all particle constructors. This changes coherent
single-orientation $M_{11}$ by $\sim$7\% (phase-sensitive), but incoherent $|J|^2$ is
identical. After orientation averaging the effect is $<0.01\%$.

\textbf{Impact:} Only \texttt{--fixed} mode affected.
Orientation averaging modes (\texttt{--random}, \texttt{--sobol}, \texttt{--adaptive}, \texttt{--auto})
always call \texttt{Rotate()} via \texttt{TracerPOTotal}, so d\_param is always initialized correctly.

See \texttt{docs/bugfix\_15x\_mismatch.pdf} and \texttt{tests/reference\_test/} for full reports and plots.

\section{New Features (March 2026)}

\subsection{Adaptive theta grid (\texttt{--auto\_tgrid EPS})}

Replaces the static zone-based theta grid with recursive bisection.
For each interval $[\theta_i, \theta_{i+1}]$: computes $M_{11}$ at the midpoint,
compares with linear interpolation. If relative error $> $ EPS, splits the interval.

\begin{itemize}
\item Minimum step: $\lambda/(8D)$ (quarter-Nyquist)
\item Maximum depth: 8 levels
\item Probe orientations: div16 physics estimate (same as adaptive start)
\item Result: 2--3$\times$ fewer theta points than static grid
\end{itemize}

Usage: \texttt{--auto\_tgrid 0.05} (5\% tolerance). Works with \texttt{--sobol}, \texttt{--random}, \texttt{--auto}.

%======================================================================
\subsection{Flag priorities}

Multiple flags can set the same parameter. Resolution order (highest priority first):

\subsubsection*{Theta grid ($N_\theta$ and theta values)}

\begin{enumerate}
\item \texttt{--tgrid FILE} — explicit non-uniform grid from file
\item \texttt{--grid T1 T2 Nphi Ntheta} — uniform grid from command line
\item \texttt{--auto\_tgrid EPS} — adaptive bisection (probes M$_{11}$)
\item \texttt{--auto} — implies \texttt{--auto\_tgrid 0.05} (bisection)
\item \texttt{--oldauto DIV} — static zone grid from \texttt{ApplyAutoThetaGrid}
\item Default: placeholder (1 theta point)
\end{enumerate}

\subsubsection*{Phi grid ($N_\varphi$)}

\begin{enumerate}
\item \texttt{--nphi N} — explicit override (highest priority)
\item \texttt{--grid T1 T2 Nphi Ntheta} — \texttt{Nphi} is the 3rd argument
\item \texttt{--auto\_phi} — formula $N_\varphi = \min(360,\; x/5 + 48)$
\item \texttt{--auto} — implies \texttt{--auto\_phi}
\item \texttt{--oldauto DIV} — same formula as \texttt{--auto\_phi}
\item Default: 1 (useless without override)
\end{enumerate}

\subsubsection*{Orientation method}

Mutually exclusive (first matched wins):
\begin{enumerate}
\item \texttt{--fixed BETA GAMMA} — single orientation
\item \texttt{--oldauto DIV} — physics-based $\beta\times\gamma$ grid
\item \texttt{--random Nb Ng} — regular grid
\item \texttt{--montecarlo N} — random uniform
\item \texttt{--orientfile FILE} — from file
\item \texttt{--auto EPS} / \texttt{--autofull EPS} — adaptive Sobol
\item \texttt{--adaptive EPS} — adaptive Sobol (manual grid)
\item \texttt{--sobol N} — fixed-count Sobol
\end{enumerate}

\subsubsection*{Beam cutoff}

\begin{enumerate}
\item \texttt{--beam\_cutoff EPS} — explicit threshold
\item \texttt{--auto}/\texttt{--adaptive} — uses their EPS as cutoff
\item Default: 0 (no cutoff)
\end{enumerate}

\subsubsection*{Coherent/incoherent}

\begin{enumerate}
\item Default: coherent (Jones summed across beams per orientation)
\item \texttt{--incoh}: incoherent (Mueller per beam)
\item \texttt{--coh\_orient}: coherent across ALL orientations (legacy)
\end{enumerate}

\textbf{Note}: \texttt{--grid T1 T2 Nphi Ntheta} — the 3rd value is $N_\varphi$ (not $N_\theta$!).
This is the original MBS convention. Use \texttt{--nphi} to avoid confusion.

\subsection{Multi-size (\texttt{--multigrid Dmin Dmax N})}

Generates $N$ sizes from $D_\mathrm{min}$ to $D_\mathrm{max}$ in logarithmic scale.
For each size: rescales particle via \texttt{Resize()}, runs full \texttt{TraceFromSobol}
(same optimized path as standalone). Results bit-exact with separate runs.

\begin{lstlisting}
# 10 sizes from D=50 to D=500, reference particle D=500
bin/mbs_po --po --sobol 1024 --multigrid 50 500 10 \
    -p 1 500 62.5 -w 0.532 --ri 1.31 0 -n 8 \
    --grid 0 180 360 180 --close -o results
\end{lstlisting}

Output: \texttt{results\_x\{N\}.dat} for each size parameter $x = \pi D / \lambda$.

\subsection{Per-beta save (\texttt{--save\_betas})}

With \texttt{--random}: writes per-beta Mueller contribution and cumulative Mueller to \texttt{\_betas/} subfolder.
Useful for convergence analysis and crash recovery.

\subsection{Parallel Phase~1 (tracing)}

Beam tracing (Phase~1) is now parallelized with OpenMP. Each thread gets its own copy of
\texttt{Particle} + \texttt{Scattering} via \texttt{CloneFor()}. Phase~1 speedup: $5.4\times$
on 12 threads.

\subsection{Additional bugfixes}
\begin{itemize}
\item \texttt{--montecarlo} missing handler config (isCoh, m\_wave, shadowOff)
\item \texttt{DiffractControlPoints}: wrong M11 formula (missing d01, d10 terms)
\item \texttt{--karczewski} ignored in fast diffraction path
\item \texttt{rdtsc} inline asm incorrect on x86-64
\item \texttt{malloc} leak in \texttt{Particle::SetFromFile}
\item OOM in \texttt{TraceAdaptive}: sub-chunked to 4096 max
\item \texttt{--grid} conflicts with auto theta/phi resolved
\item \texttt{--oldauto} N rounding: ceil instead of ODD (smoother size scans)
\item \texttt{--point} crash from uninitialized handler pointer
\item MPI missing \texttt{\}} in \texttt{TraceAdaptive} convergence block
\end{itemize}
